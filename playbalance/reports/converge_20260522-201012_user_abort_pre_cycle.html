💬 이번 세션 대화 요약

왜 이 KPI 였나

사용자가 보수 KPI Dur >= 90 으로 선택. baseline ~62s 대비 90s 도달만 해도 burst 축소 효과 검증으로 충분하다는 합의. /bal-plan 의 권장 명령 그대로 실행.

사전 가설

  • Ghost.asset Pattern=Dense Count=30 의 60s burst 가 structural cutoff 의 메커니즘
  • spawn_burst_count knob (신규 추가) 으로 Count 30→15 하면 burst 축소 → Dur 증가

사용자가 표명한 우려

  • spawn_rate (BaseRatePerSecond) 는 Random 패턴 한정이라 Ghost(Dense) 에 무효였던 점은 이전 cycle 들에서 확인됨
  • 정수 round-trip 흡수 (enemy_damage 작은 정수에 ±15% → no-op) 메모리 트랩

※ 이 박스는 Claude가 종료 시점에 대화 로그를 요약한 재구성입니다. 정확한 사용자 발언은 대화 원문을 참조하세요.

📊 데이터 기반 자동 분석

1) 시작 상태 (baseline)

5 plays 중 play 4 가 38s 에 단명, 나머지 4 plays 는 62±1s 의 익숙한 60s cutoff 패턴. 평균 57.57s, stddev 10.61s, σ/μ=18.4%. play 4 가 outlier 로 작용해 통계가 흩어졌다.

핵심 관찰: diagnostic 6개 중 structural_duration_cutoff 만 spawn_burst_count knob 의 trigger. 그런데 σ/μ=18.4% > 임계값 10% 라서 이 diagnostic 이 발화하지 않음. 따라서 이번 cycle 의 핵심 가설 (spawn_burst_count 로 Ghost Count 30→15 적용) 검증 자체가 시작도 못 함. 대신 killer_concentration (Gear 60%), level_cap_low (lvl 2.8), high_damage_taken_variance (σ/μ=0.89) 가 발화 → 기존 stat knob (enemy_damage Gear, level_up_xp_increment, player_max_hp) 만 매칭됨. cycle 1 과 동일한 구성.

4) 진행 신호

  • play 4 단명 (38s, lvl 1) 가 1/5 표본에서 σ 를 크게 키움. N 을 늘려 outlier 영향 희석하면 σ/μ 가 임계값 아래로 내려갈 가능성.
  • enemy_damage Gear: Int 3 → 3 (정수 round-trip 흡수) — 메모리 트랩이 실시간 재확인됨.
  • 사망 원인 (Killer) 분포: Gear 60% / Slime 40%. NearEnemy (사망 시 가까운 적) 는 Gear 80% / Ghost 20%. 데이터상 Ghost 는 사망의 직접 원인은 아니나 60s burst 후 환경 압박은 명확.
