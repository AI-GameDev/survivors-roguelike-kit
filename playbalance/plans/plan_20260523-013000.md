## 3. Hypothesis priority

1. **60s 동시 spawn cliff 가 진짜 sustained 압력 원인** — 데이터로 입증 (사망 시점 활성 적 150마리, 60s 에 spawn 17→55 폭증). 4개 set StartTime 분산이 가장 직접적 lever.
2. **Stage 1 단독으로 KPI 도달 가능** — spawn 시점만 30s 늦춰도 80s wall → 110s wall 로 이동. 모델이 이미 60s wall 뚫는 강함으로 110s wall 도 뚫을 가능성. 평균 150~200 예상.
3. **Stage 2 미세 수렴은 양방향 필요** — 사용자 의도 "무조건 오래 버티지 못하게". 현재 bal-apply.json 모든 knob 이 약화 방향. overshoot 시 manual 강화 또는 config 보강.

## 4. Traps (주의)

- **모델 학습 분포 외**: 모델 v17 은 60s wall 환경 학습. StartTime 변경은 학습 분포 밖. 모델이 그 구간을 어떻게 다룰지 미지수 — 의외로 잘 적응 (탐험적 행동) 또는 stall 가능. 한 사이클 후 v18 재학습 고려.
- **EndTime 함정**: Ghost 의 EndTime=60 은 StartTime 과 같아 1회 burst. StartTime 만 옮기면 EndTime < StartTime 되어 invalid. **둘 다 같이 옮길 것** (StartTime 60→90, EndTime 60→90).
- **bal-apply derived target trap**: spawn_burst_count knob 은 사망 직전 5초 modal spawn 잡아 Gear (Random) 매칭. Stage 2 에서 또 no-op 발생 가능. Stage 2 는 `--auto` 대신 `knob=enemy_damage` 명시 권장.
- **enemy_damage 정수 round-trip**: Gear Attack=3. -15% no-op. Stage 2 에서 이 knob 쓰려면 절대값 직접 (3→2) 또는 adjust_default 키워야.
- **bal-apply.json 양방향 부재**: knob 전부 약화 방향. Stage 1 이 너무 강하면 자동 재조정 불가.
- **PlayTrace 시간 단위**: client_time 은 ms. PlayTrace dashboard 의 시간축 단위 확인 필수 (대부분 sec).
