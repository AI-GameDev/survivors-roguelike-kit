## 2. Direction

### KPI

```
Dur 180~270 AND cause=death
```

이유: 사용자 목표 평균 생존 3분~4분30초 (180~270s). `cause=death` 조건으로 timer cap (442s 같은 outlier) 자동 제외 → mean 안정성 + 자연사망 회귀를 동시에 강제.

### Baseline

**새로 측정** (N=5~7). `/bal-converge` 가 자체 측정. `_026` 은 σ/μ=85% 라 임계 초과 (memory: `feedback_baseline_stability_precondition`).

### Knob whitelist

허용 (3개):
- `spawn_burst_count` — Burst spawn count (Dense/Circle). 직전 pass 의 결정적 lever. Count=15 가 너무 약해 timer cap hit (1/5) 발생 → Count 18~22 후보로 미세조정.
- `enemy_hp` — Enemy SO HP. 적이 약하면 kill snowball ↑ → Dur ↑.
- `level_up_xp_increment` — 레벨업 가속화. player snowball ↑ → Dur ↑.

제외 (이유):
- `enemy_damage` — 정수 round-trip 흡수 (memory: 작은 정수 자산에 ±15% no-op). 다음 사이클에서 adjust_range 키운 뒤 재시도 가능.
- `spawn_rate` — Dense/Circle 패턴은 BaseRate 무효, Count 가 단일 lever (memory: `project_spawnset_pattern_lever`). `spawn_burst_count` 가 진짜 lever.
- `spawn_intensity_curve` — AnimationCurve, 수동 편집 대상.
- `player_max_hp` — 플레이어 직접 강화는 ML 모델 행동 학습된 환경과 mismatch 위험. 이번 사이클은 적/진행 lever 우선.

→ 사용자가 위 제외 항목 중 일부를 다시 포함시키고 싶으면 `.claude/bal-apply.json` 의 knob `auto` 필드를 직접 토글해야 함 (bal-converge 인자로 whitelist 전달 불가).

## 3. Hypothesis priority

1. **`spawn_burst_count` 미세조정 (Count 15 → 18~22)** — 직전 사이클의 입증된 lever. timer cap rate 1/5 → 0/5 회귀, σ 축소 기대. mean 은 164s 에서 다소 내려갈 수 있지만 KPI 하한 (180s) 통과 위해선 다른 lever 가 보완해야 함.
2. **`enemy_hp` 약화 (-15~-30%)** — kill snowball 가속화. Lvl 상승 → 자기방어력 ↑ → Dur ↑. Count 강화 효과를 상쇄.
3. **`level_up_xp_increment` 감소 (-15~-30%)** — 레벨업 가속화. 2번과 같은 방향, snowball 보조.

→ bal-apply 가 이 3개 lever 의 적절한 조합을 자동 탐색. KPI 가 좁은 구간 (180~270) 이라 1~2 iter 안에 정확 hit 어려우면 max_iter=3 으로 안전망.

## 4. Traps (주의)

- **enemy_damage 정수 round-trip 흡수** (memory: `project_ghost_burst_structural_cutoff` 의 trap 노트) — whitelist 제외했지만 사용자가 직접 포함시킬 경우 ±15% 가 no-op 될 수 있음. round-trip 검증 필수.
- **SpawnSet 패턴-knob 매핑** (memory: `project_spawnset_pattern_lever`) — Dense/Circle 패턴 자산은 BaseRate 무효, Count 가 lever. `spawn_burst_count` knob 은 이 차이를 이미 흡수하지만 새 자산 추가 시 재확인.
- **policy mismatch 누적** — 모델 v17 은 Count=30 환경으로 학습됐고 이번에 Count 한 번 더 바뀌면 변화 누적. timer cap hit rate 가 다시 증가하면 v18 재학습이 다음 단계.
- **PlayTrace size 함정** (memory: `feedback_playtrace_size_limit`) — bal-converge 의 metric fetch 가 sparse 키를 잘라 0건 오진단 가능. 사이클 중 의심되면 per-key fetch (`key=episode.cause` 등) 로 재확인.
