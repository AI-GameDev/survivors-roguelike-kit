## 3. Hypothesis priority

1. **spawn timing 이 cutoff 의 진짜 원인**. baseline σ/μ=0.2% 는 단순 stat 약함이 아니라 t=60 부근의 burst spawn 사건 (Ghost 변종 7개) 이 dominant 인 신호. `spawn_rate` 인하 + `spawn_intensity_curve` 도 수동 조정 후보로 본다.
2. **`level_up_xp_increment` 인하로 레벨업 가속** → 플레이어 스케일링이 spawn 강화를 따라잡을지. 1번 효과가 부분적이면 보조.
3. **`enemy_damage` 는 효과 약함이 확정** (round-trip). 그래도 whitelist 에 두는 건 다른 dimension 의 노이즈/시그널 분리 용도.

## 4. Traps (주의)

- **enemy_damage round-trip**: 자산 값이 작은 정수 (3) 라 ±15% 가 반올림으로 흡수 → no-op. 이번 사이클에서도 반복될 가능성 높음. iter 진행 도중 적용 로그에서 `(+0.0%)` 가 보이면 same trap. min absolute step 옵션 도입은 별개 후속 항목.
- **Ghost burst spawn 60s cutoff**: stat 만 조정해선 못 뚫는다는 사실은 이미 baseline σ 가 증명함. spawn_rate 가 1순위 후보지만, 효과가 약하면 spawn_intensity_curve 수동 편집이 진짜 답일 수 있음. 사이클 후 미달이면 그쪽으로.
- **smoke 제약 vs 본격**: bal-apply 의 `min_runs_for_analysis=3` 때문에 N=2 였다면 iter 2 진입 시 자동 abort. 이번엔 N=5 라 안전.
- **per_play_timeout vs Dur 목표**: KPI 가 180~240s 를 목표로 하지만 config default `per_play_timeout=10m` 라 충분. 만약 사용자가 별도 인자로 줄이면 안 됨.
